3. Vue.js 의 장점
	직관적, 배우기쉽다
	재사용을 통한 어플리케이션 개발기간을 단축, 양질의 코드를 생산할 수 있음
	Angular -> React -> Vuejs 순으로 개발되었기 때문에 
	Angular의 장점(데이터바인딩) 과 React의 장점(가상돔) 을 모두 가지고 있음
	

+) Angular의 데이터 바인딩(two-way 데이터 바인딩)
	데이터와 웹 화면의 요소(html dom)가 서로 양방향으로 연결되어 있어서 
	어느 한쪽에서 변경이 일어나게 되면 연결되어 있는 다른쪽에 자동으로 반영되는
	것을 의미
	
+) React
	Dom의 문제 DOM트리구조를 모두 갱신하는 비효율성으로 인해 치명적인 성능저하
	극복 -> virtualDom 제공 

=> Vue.js는 Angular 가 가지고 있는 데이터 바인딩의 장점과 React가 가지고 있는
	virtual DOM의 장점을 모두 수용해서 개발이 완료된 프레임워크
	

4. Vue.js 특징
1) MVVM 패턴사용
	model-view-viewmodel 을 의미한다
	
	view : 화면, dom으로 
	model : 서버와 통신, 일반 js객체
	view model: view와 model 사이 연결, domlistener,  vue
	
	view -> viewModel(Dom Listener) -> model
	view <- viewModel(directive)<- model

	view - view model 사이에서 데이터바인딩발생
	
	

	어플리케이션 로직과 UI의 분리를 위해 설계된 패턴이다
	viewmode은 model의 데이터변경에 대한 알림을 받고 바인딩 기법을 통해
	미리 정해진 템플릿 규칙에 따라 view를 다시 그려낸다
	viewModel : 살제 논리 및 데이터 흐름을 담당
	

2) 컴포넌트
	작은 단위부터 특정 기능을 처리하는 더 큰 단위까지 화면(view)를 이루고 
	있는 작은 단위의 view 중에 다른 화면에서도 사용되는 view가 있다
	단위의 view를 재사용할 수 있는 구조로 개발하는 것을 의미
