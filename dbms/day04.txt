0. 복습
SQL문(쿼리문)의 종류
- SQL 명령어의 성격에 따른 분류
	DDL : 데이터 정의 언어
		테이블 조작 또는 제어 관련 쿼리문
	DML : 데이터 조작 언어
		데이터를 조작하는 쿼리문
	DCL : 데이터 제어 언어(권한)
	TCL : 트랜잭션 제어 언어

- DML 
	SELECT : 데이터 조회
	INSERT : 데이터 추가
	UPDATE : 데이터 수정
	DELETE : 데이터 삭제

- SELECT
	테이블의 데이터를 조회한다
	SELECT 컬럼명
	FROM 테이블명
	[WHERE 조건식];
- INSERT
	테이블에 데이터를 삽입한다
	
	INSERT INTO 테이블명
	VALUES(값,...);

	INSERT INTO 테이블명 (컬럼명, ...)
	VALUES(값, ...);
- UPDATE
	테이블에 데이터를 수정한다
	UPDATE문에 조건문을 입력하지 않으면 모든 데이터가 수정된다

	UPDATE 테이블명
	SET 컬럼명1 = 값1, ...
	[WHERE 조건식];
- DELETE
	테이블에 데이터를 삭제한다
	DELETE문에 조건문을 입력하지 않으면 모든 데이터가 삭제된다
	
	DELETE FROM 테이블명
	[WHERE 조건식];

키의 종류
	1. 기본키(Primary Key)
		테이블 내의 각 행을 고유하게 식별하는데 사용
		중복 허용하지 않는다
		Null 값도 허용하지 않는다
	2. 외래키(Foreign Key)
		외래키를 사용하여 두 테이블 간의 관계를 설정하고 데이터 무결성을 유지
	3. 조합키(복합키)
		두개 이상의 컬럼의 하나의 key로 지정하는 것
		모든 테이블은 하나의 PK만 존재하지만, 조합키를 사용하여 두개 이상의 컬럼을 하나의 PK로 사용할수있다
	4. 후보키(Candidate Key)
		기본키가 될 수 있는 값(기본키의 후보)
	5. 대체키(Alternate Key)
		후보키에서 기본키로 선택되지 않은 후보키들
	6. 슈퍼키(Super Key)
		레코드를 고유하게 식별하기 위한 속성 또는 속성의 조합
		슈퍼키는 후보키와 대체키의 상위개념

제약조건
	1. PRIMARY KEY(PK)		기본키
		고유한 값이며 각 행의 구분점으로 사용된다
		중복이 없고 NULL값을 허용하지 않는다
		ex) 회원번호(1, 2, 3, 4) User 테이블의 기본키가 된다

	2. FOREIGN KEY(FK)		외래키
		다른 테이블의 PK를 사용하며 중복이 가능하다
		보통 테이블끼리의 관계를 맺을 때 사용한다
		NULL값을 허용하지만 좋은 상황은 아니다
		ex) Board 테이블의 회원번호(FK)가 User 테이블의 회원번호(PK)를 참조한다

	3. UNIQUE KEY(UK)		유일키
		NULL값은 허용하지만 중복을 허용하지 않는다
		ex) User 테이블의 아이디 컬럼은 유일키로 설정되어 중복된 아이디를 허용하지 않는다
	
	4. CHECK(CK)
		컬럼의 값이 특정 조건을 만족해야한다(값의 범위나 조건 사용)
	
	5. NOT NULL
		해당 컬럼의 값은 NULL값이 허용되지 않는다
	
	6. DEFAULT
		해당 컬럼의 값이 입력되지 않을 경우, 기본값으로 자동 입력된다
		지정하지 않을 경우 NULL로 자동 사용된다

3. DDL(Data Definition Language) : 데이터 정의어
	테이블 조작, 제어 관련 쿼리문

CREATE : 테이블 생성
	CREATE TABLE 테이블명(
		컬럼명 자료형(용량) [제약조건], 
		...
	);

DROP : 테이블 삭제
	DROP TABLE 테이블명;

ALTER : 테이블 수정
	ALTER TABLE 테이블명
		- 테이블명 수정 : RENAME TO 새로운 테이블명;
		- 컬럼 추가 : ADD(새로운 컬럼명 자료형(용량));
		- 컬럼명 변경 : RENAME COLUMN 기본 컬럼명 TO 새로운 컬럼명;
		- 컬럼 삭제 : DROP COLUMN 기존 컬럼명;
		- 컬럼 타입 수정 : MODIFY(기존 컬럼명 자료형(용량));

TRUNCATE : 테이블 내용 전체 삭제
	TRUNCATE TABLE 테이블 명;
